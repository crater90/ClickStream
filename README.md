# ClickStream

A full-stack, self-hosted web analytics platform. Drop a single `<script>` tag on any webpage to capture page views and clicks, then explore sessions and heatmaps in a real-time dashboard.

```
event-tracker/
├── backend/        Node.js · Express · MongoDB
├── dashboard/      Next.js 14 · Tailwind · TypeScript
└── analytics/   Vanilla JS tracking script + demo page
```

### Data Flow in Plain English

1. **Tracking** — `tracker.js` is embedded in any webpage. It assigns each browser a UUID session ID (stored in a cookie + localStorage). Every page view and click is queued in memory and flushed to the backend every 500 ms via `fetch` with `keepalive: true`.

2. **Ingestion** — The Express backend receives events, validates them, and writes them to MongoDB via Mongoose. Batch inserts (`/api/events/batch`) are used for efficiency.

3. **Storage** — All events land in a single `events` collection. Compound indexes make the two heaviest queries (session journeys and heatmaps) fast even at scale.

4. **Dashboard** — Next.js fetches from the backend on the client side (`"use client"` components). Sessions are aggregated server-side in MongoDB using `$group`. Heatmap coordinates are normalised to a fixed canvas size so clicks from different screen resolutions land in the same proportional position.

## Local Setup

### Prerequisites

| Tool | Version | Check |
|---|---|---|
| Node.js | 18 + | `node -v` |
| npm | 9 + | `npm -v` |
| MongoDB Atlas account | free M0 | [cloud.mongodb.com](https://cloud.mongodb.com) |

You need **three terminals** — one per package.

---

### Terminal 1 — Backend

```bash
cd backend

# Install dependencies
npm install

# Create your .env file
cp .env.example .env
```

Open `.env` and set your MongoDB Atlas connection string:

```env
PORT=3001
MONGODB_URI=mongodb+srv://<user>:<password>@cluster0.xxxxx.mongodb.net/event_tracker?retryWrites=true&w=majority
```

```bash
# Start the server
npm run dev
```

Expected output:
```
✅ MongoDB connected
🚀 Server running on http://localhost:3001
```

Verify it's healthy:
```bash
curl http://localhost:3001/health
# {"status":"ok","mongodb":"connected","uptime":3}
```

---

### Terminal 2 — Dashboard

```bash
cd dashboard

# Install dependencies
npm install

# Start Next.js dev server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). The dashboard will show empty state until events are recorded.

---

### Terminal 3 — Tracker Demo

> ⚠️ **Must be served over HTTP**, not opened as a `file://` URL — browsers block cross-origin `fetch()` from `file://`.

```bash
cd tracker-demo

npx serve .
# Serving at http://localhost:3000  (or another free port)
```

## API Reference

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/api/events` | Store a single event |
| `POST` | `/api/events/batch` | Store multiple events at once |
| `GET` | `/api/events/sessions` | List all sessions with aggregated counts |
| `GET` | `/api/events/sessions/:id` | Full ordered event journey for one session |
| `GET` | `/api/events/heatmap?page_url=` | Click coordinates for a specific page |
| `GET` | `/api/events/pages` | All tracked pages with view/click counts |
| `GET` | `/health` | API health check |

### Event object

```jsonc
{
  "session_id":      "b3f1a2c4-...",   // UUID, auto-generated by tracker.js
  "event_type":      "page_view",      // "page_view" | "click"
  "page_url":        "https://example.com/pricing",
  "timestamp":       "2025-05-15T10:30:00.000Z",
  "click_x":         640,              // pixels from left  (click only)
  "click_y":         380,              // pixels from top   (click only)
  "viewport_width":  1440,             // browser viewport  (click only)
  "viewport_height": 900,              // browser viewport  (click only)
  "referrer":        "https://google.com"
}
```

## Embedding the Tracker on Your Own Site

```html
<!-- Add to the <head> or bottom of <body> -->
<script
  src="https://your-cdn-or-path/tracker.js"
  data-api="https://your-backend.railway.app/api/events">
</script>
```

JavaScript API:

```js
// Read the current session ID
window.EventTracker.getSessionId();   // "b3f1a2c4-..."

// Force-flush the event queue immediately
window.EventTracker.flush();

// Inspect the pending queue
window.EventTracker.getQueue();       // [{ event_type: "click", ... }]
```

The script automatically handles:
- SPA navigation (`pushState` / `popstate`)
- Page unload (synchronous XHR fallback)
- Cookie + localStorage session persistence
- `text/plain` content-type fallback for `sendBeacon`
